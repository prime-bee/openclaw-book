Início/Capítulos/Capítulo 2: Arquitetura

Veja como gateway, clientes, nós, sessões e eventos se encaixam na arquitetura operacional do OpenClaw.

Capítulo 2 de 22Leitura guiadaReferências oficiais ao final

Como as peças se encaixam

A arquitetura descrita na doc gira em torno de um princípio: um gateway centralizado controla superfícies de mensagem, clientes de controle, nós e sessões.

Componentes principais

  • Gateway: processo persistente que mantém conexões, expõe APIs e serve como fonte de verdade.
  • Clientes: CLI, interface web e outros consumidores do plano de controle.
  • Nodes: dispositivos conectados via WebSocket com capacidades extras como câmera, canvas e localização.
  • Agentes e sessões: contexto conversacional isolado, com memória e ferramentas.

Fluxo básico

Mensagem entra → Gateway recebe → sessão é resolvida → agente roda → eventos/stream saem → resposta volta ao canal

O detalhe importante: sessões

OpenClaw não trata tudo como um bloco único. Há sessões separadas, o que permite:

  • contextos independentes
  • agentes com workspaces diferentes
  • memória localizada
  • roteamento por chat, canal ou usuário
Boa intuição: o gateway é o cérebro operacional do sistema; a sessão é a memória de curto prazo de cada conversa.

Primeiro frame: connect

Pela documentação de arquitetura, clientes e nós falam com o gateway por WebSocket. O primeiro frame precisa ser connect. Depois disso vêm requests, responses e events.

Exemplo mental

Cliente → connect
Gateway → hello-ok
Cliente → req:agent
Gateway → accepted
Gateway → eventos de agent (stream)
Gateway → resposta final

Exercício

Pergunta: por que faz sentido existir um gateway único em vez de cada canal falar diretamente com um modelo?

Checklist de arquitetura saudável

  1. o gateway responde a status e health sem latência estranha
  2. cada canal aponta para o agente esperado
  3. sessões são fáceis de inspecionar e entender
  4. eventos e logs permitem reconstruir o que aconteceu

Se um desses pontos falha, o problema costuma parecer “aleatório” mesmo quando a arquitetura está apenas mal observada.
