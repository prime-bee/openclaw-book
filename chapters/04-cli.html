A CLI do OpenClaw

A CLI é ampla. Em vez de decorar tudo, vale aprender os grupos principais.

Setup

setup, onboard, configure

Operação

status, health, logs, gateway

Chat e agentes

message, agent, agents, sessions

Ambiente

channels, models, memory, skills

Comandos para decorar primeiro

openclaw status
openclaw gateway status
openclaw channels status --probe
openclaw models status
openclaw logs --follow
Heurística útil: a CLI fica mais fácil quando você separa os comandos em três perguntas: "o sistema está vivo?", "o subsistema certo está saudável?" e "consigo agir com segurança?".

Mapa mental de uso diário

Necessidade
Grupo de comandos
Entender estado geral
status, health, logs

channels, gateway, sessions

agent, agents, memory

models, skills, channels

Exemplos práticos

1) Ver canais

openclaw channels list
openclaw channels status --probe

2) Rodar uma mensagem de agente

openclaw agent --message "Resuma meus arquivos de memória"

3) Ver memória indexada

openclaw memory status
openclaw memory search "preferências do usuário"

Fluxo de troubleshooting pela CLI

  1. Comece com openclaw status para validar saúde geral.
  2. Desça para a área específica: gateway, channels, models ou memory.
  3. Use logs e flags de probe/deep quando o status simples não basta.
  4. Só depois faça ações mutáveis como restart, add ou reconfigure.

Dica didática

Use a CLI como se fosse uma árvore:

openclaw <área> <ação>

Exemplos:
openclaw gateway restart
openclaw models list
openclaw channels add

Atalhos de aprendizado

Para iniciantes

Decore poucos comandos, mas execute-os de verdade até associar nome, saída e propósito.

Para operação diária

Mantenha um pequeno repertório de status, probe e logs antes de partir para restart.

Para equipes

Padronize uma sequência de diagnóstico comum para que todo mundo investigue incidentes do mesmo jeito.

Exercício

Prática: escolha três comandos que você provavelmente usaria toda semana e escreva por que cada um deles importa.
