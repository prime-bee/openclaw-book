Memória no OpenClaw: arquivos primeiro

A doc oficial é bem clara: a memória do OpenClaw é Markdown no workspace. O modelo só retém de forma confiável o que foi escrito em disco.

As duas camadas principais

  • memory/YYYY-MM-DD.md — log diário, mais cru
  • MEMORY.md — memória longa, mais curada
Resumo honesto: se não escreveu, não existe como memória persistente.

Ferramentas de memória

  • memory_search: busca semântica
  • memory_get: leitura pontual de trechos

Quando escrever memória

  • preferências duráveis do usuário
  • decisões importantes
  • contexto recorrente
  • coisas que você explicitamente pediu para lembrar

Workspace

O workspace é mais do que uma pasta. É o território operacional do agente: documentação, identidade, memória, scripts e materiais auxiliares.

workspace/
  AGENTS.md
  SOUL.md
  USER.md
  IDENTITY.md
  MEMORY.md
  memory/
  ...

Exemplo prático

Se o usuário diz “sempre me responda primeiro em português”, isso pode virar memória longa. Se hoje vocês decidiram o nome de um projeto, isso pode ir para a memória diária.

Exercício

Faça a classificação: o que vai para MEMORY.md e o que vai para memory/2026-03-19.md?

1. “O usuário prefere respostas curtas.”
2. “Hoje foi criado o avatar do agente.”
3. “O usuário usa Telegram como canal principal.”
